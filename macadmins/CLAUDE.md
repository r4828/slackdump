# macadmins archive — Claude Code guidance

This directory holds an offline, read-only snapshot of selected channels from
the public `macadmins.slack.com` community workspace, plus tooling to refresh
it. The purpose is to let Claude (and other agents) answer Mac/MDM questions
grounded in actual community discussion.

## What is here

- `channel_names.txt` — human-edited list of channel names to include.
- `build_channel_list.sh` — resolves those names to Slack channel IDs/URLs
  via `slackdump list channels`, producing `channels.txt`.
- `channels.txt` — generated list of `https://macadmins.slack.com/archives/...`
  URLs, fed to `slackdump archive` as positional args.
- `archive/` — slackdump output. Contains `slackdump.sqlite` and, if files
  were enabled, `__uploads/` and `__avatars/`. Ignored by git.
- `md/` — per-channel Markdown rendered from `archive/slackdump.sqlite`
  (generated by `slackdump_to_md.py`). Ignored by git.
- `slackdump_to_md.py` — SQLite → Markdown converter.
- `.mcp.json` — Claude Code MCP server config pointing at `archive/slackdump.sqlite`.

## MCP tools available (via the `slackdump-macadmins` server)

All read-only:

- `list_channels` — channels present in the archive.
- `get_channel` — channel metadata by ID.
- `list_users` — workspace users.
- `get_messages` — paginated messages (`channel_id`, `limit`, `after_ts`).
- `get_thread` — parent + replies (`channel_id`, `thread_ts`).
- `get_workspace_info` — team/user metadata.
- `load_source` — swap to a different archive at runtime.
- `command_help` — slackdump CLI help text.

## Refresh workflow

```bash
# once per machine — authenticate
slackdump workspace new macadmins

# rebuild the URL list and re-archive
./build_channel_list.sh
slackdump archive \
  -workspace macadmins \
  -files=false -avatars=false -channel-users \
  -o ./archive \
  @channels.txt

# re-render Markdown
python3 slackdump_to_md.py ./archive/slackdump.sqlite -o ./md
```

To resume an interrupted archive, run `slackdump resume ./archive` — the
SQLite `SESSION.FINISHED=0` row lets slackdump continue where it left off.

## Conventions for agents

- Prefer the MCP tools for live queries; they always reflect the current
  archive.
- Do not modify `archive/slackdump.sqlite` directly. Re-run the archive
  command to update it.
- When summarising or quoting from channels, cite the channel name and an
  approximate date (from `get_messages` output `ts`) so humans can verify.
